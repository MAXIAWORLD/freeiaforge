C'est quoi
Un proxy IA gratuit sur ton PC
FreeIA Gateway route tes requêtes vers les meilleurs LLMs gratuits disponibles. Si un provider tombe ou atteint sa limite, le circuit breaker bascule automatiquement sur le suivant — sans interruption.

Tu obtiens une API locale compatible avec n'importe quel outil qui supporte un endpoint OpenAI custom (Cursor, Continue.dev, AnythingLLM, LibreChat…). Le cache sémantique fastembed évite de reconsommer du quota sur des questions déjà posées.

Fonctionnalités
Tout ce qu'il te faut
Conçu pour fonctionner localement, sans abonnement, sans VPS.
⚡
Fallback + circuit breaker
Cerebras → Groq → Gemini → … → Ollama. Circuit breaker 3 états (CLOSED/OPEN/HALF_OPEN) : 3 erreurs → blacklist 10 min → test récupération automatique.
📈
Quota manager dynamique
Suivi des requêtes et tokens par provider en SQLite. Les providers les plus "pleins" passent en premier. Reset daily automatique. Limites configurables via providers.json distant.
🔗
API compatible OpenAI
Endpoint /v1/chat/completions avec streaming SSE natif. Headers X-FreeAI-Provider et X-FreeAI-Task. Branche n'importe quel outil OpenAI-compatible.
🧠
Cache sémantique
fastembed all-MiniLM-L6-v2 (80 MB, CPU-only, zéro torch). Similarité cosine ≥ 0.90 → réponse instantanée, 0 quota cramé. TTL configurable, SQLite local.
🎯
Task routing
Détection automatique : vision (image_url) → Gemini/OpenRouter, long context (>24k chars) → Gemini 1M, code (```) → Groq/Cerebras en priorité.
🔐
100% local · privacy
Rien ne passe par nos serveurs. Dashboard HTML auto-refresh sur :8002. Historique 7 jours en SQLite. Ollama en fallback si tous les clouds tombent.

Installation
Prêt en 1 commande
Prérequis : Docker. Ou Python 3.12 si tu préfères le mode manuel.
1
Lancer avec Docker (recommandé)
Une seule commande — image publiée sur Docker Hub, mise à jour automatiquement.
docker run -p 8002:8002 \ -e CEREBRAS_API_KEY=xxx \ -e GROQ_API_KEY=xxx \ -e GEMINI_API_KEY=xxx \ maxiaworld/freeiaforge:latest
2
Ou : wizard interactif (mode manuel)
Clone le repo et lance le wizard — il guide la configuration clé par clé.
git clone https://github.com/MAXIAWORLD/freeiaforge cd freeiaforge # Linux / macOS ./setup.sh # Windows setup.bat
3
Connecter ton client IA
Dans Cursor, Continue.dev, AnythingLLM ou LibreChat — configurer le provider OpenAI custom :
Base URL : http://localhost:8002/v1 API Key : (laisse vide ou mets n'importe quoi) Model : freeai-gateway
4
Dashboard local
Ouvre http://localhost:8002 — état des providers, quota restant, historique 7 jours, circuit breaker status. Auto-refresh toutes les 30 secondes.
Stack technique
Simple, robuste, extensible
Tout en Python 3.12. Aucune dépendance cloud. Zéro torch. Fonctionne hors ligne dès que les clés sont configurées.
✓
FastAPI + Uvicorn — middleware ASGI pur (streaming-safe), SSE token par token, format OpenAI strict, headers X-FreeAI-Provider
✓
Providers directs — 9 implémentations natives (httpx async), aucun wrapper tiers, normalisation vers format OpenAI pour chaque provider
✓
SQLite 3 fichiers — quota.db (quotas + circuit breaker), cache.db (cache sémantique), stats.db (historique 7 jours)
✓
fastembed all-MiniLM-L6-v2 — 80 MB, CPU-only, zéro torch. Cache sémantique cosine ≥ 0.90, TTL 1h, SQLite local
✓
346 tests pytest · CI GitHub Actions — TDD strict, tous verts avant chaque release. Docker Hub auto-publish sur tag v*.*.*
✓
Multi-clés + credential pool — rotate GROQ_API_KEYS=k1,k2,k3, cooldown 24h sur 401/429 par clé, persisté en SQLite
